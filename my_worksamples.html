Work Sample: Quick Quiz Android App

As an assignment for my Android Development course, I was asked to create an app that took a .txt file containing a comma-separated list of questions and answers and transform it into a multiple-choice quiz that allows the user to choose the correct answer to a question from a randomly-generated list of responses. The app also keeps track of the user name, number of questions remaining in the quiz, and the user's current score.

This app required a broad range of Android knowledge, from using hashmaps to create the required collection of answers and questions to creating intents to carry user info between activities. Ensuring that the selection of possible answers was random enough while still containing the correct answer required some creativity. I researched various randomizing functions before settling on shuffle() as the cleanest option. For each question, I created an arrayList containing all possible answers, located the correct answer for the question using the hashmap, then removed it from the arrayList. From there, it was as simple as shuffling the arrayList and adding three other answers as options to display. Looking at it now, the code seems simple, but at the time I was very proud of making it work with (what I thought was) such elegant code.

Creativity is important when working in Android, both for thinking about the logic of the program and for designing the UI. I am pretty pleased with the color palette of this app, but in the future I would like to smooth out the rest of the UI. Better buttons and a more interesting scoring layout would also improve the feel.

Work Sample: Bus Tracker - GeoJSON and Leaflet.js

This app was created to fetch real-time tracking data from the HRM Bus API, convert it into GeoJSON format and use Leaflet.js to map the results, real-time, on a map. The original goal of the app was to track all buses on routes 1-10, but I altered the original design to show only the routes that I used to use at my old job. The user can select a marker to determine route number and the current speed of the selected bus.

This project needed an understanding of JSON and GeoJSON data and how to use fetch and array functions. We were not allowed to use loops of any kind for this app, so I needed to learn GeoJSON functions like OnEachFeature to add multiple customized popups to the appropriate layers.

This is one of the most practical assignments that I completed at NSCC. It taught me how to handle, format and transform GeoJSON data. More importantly, it opened up the world of API's to me, and what they can do with relatively simple code. I'm currently working on another project using a different JSON API, and I can see that understanding how to work with the data available out there is one of the most important things that I've learned at NSCC.
